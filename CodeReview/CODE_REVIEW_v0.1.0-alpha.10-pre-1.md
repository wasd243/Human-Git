# Code Review — `v0.1.0-alpha.10-pre-1`

## Verdict
**Needs changes before release.** The architecture is coherent and the Rust/TS boundary is clean, but there are a few behavior risks that can lead to noisy runtime failures or incorrect Git history.

## High-impact findings

| Severity   | Finding                                                               | Evidence                                                                                                                                                                                                                                    | Why it matters                                                                                                | Recommendation                                                                                                               |
|------------|-----------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
| **High**   | **Quick Deploy can create empty commits**                             | `src\modules\operations\commit_and_push.rs:69-103` stages/writes tree and commits without checking whether there are actual changes.                                                                                                        | If this command is triggered outside current UI guardrails, history can be polluted with meaningless commits. | Add a backend-side guard (e.g., compare index/tree to HEAD or check status) and return a clear “no changes to commit” error. |
| **Medium** | **Stale cached repo path can put watcher into repeated failure loop** | `src\app\desktop\ts\gui.ts:173-183` restores cached path automatically; `src\modules\operations\open_folder.rs:14-48` accepts path without existence/access checks; `src\modules\watcher.rs:182-188` retries watch after failure in a loop. | A moved/deleted folder leads to recurring watch errors and degraded UX/log noise at startup.                  | Validate path existence/readability in `update_repo_path` and clear invalid cache entries on restore failure.                |
| **Medium** | **Periodic refresh can overlap and issue concurrent backend calls**   | `src\app\desktop\ts\gui.ts:208-212` uses `setInterval(async ...)` every 1s without in-flight lock.                                                                                                                                          | Slow `refreshFileList()` calls can stack, causing unnecessary load and intermittent UI jitter.                | Add a simple in-flight flag or switch to a self-scheduling `setTimeout` loop.                                                |
| **Low**    | **Remote URL validation is stricter than Git itself**                 | `src\app\desktop\ts\modules\buttons.ts:77` requires `.git` suffix.                                                                                                                                                                          | Valid remotes like `git@github.com:owner/repo` (without `.git`) are rejected.                                 | Relax validation to accept standard Git URL forms without mandatory `.git`.                                                  |

## Positive notes

- Good use of `spawn_blocking` in Tauri handlers for blocking Git operations (`src\modules\ui_bridge\handlers.rs`).
- Error messages are generally explicit and actionable across Git flows (`pull.rs`, `push.rs`, `fetch.rs`, `remote.rs`).
- Frontend module decomposition is clear (`buttons.ts`, `listener.ts`, `refreshFileList.ts`), making UI behavior easy to trace.
